# Obsidian 笔记规范

基于轻量 PARA 结构的 Obsidian 笔记体系，文件夹做大类区分，细节靠 Properties 属性组织，链接串联一切。

## 关键点

- **文件夹只做大类，细节靠属性**：`20_笔记/领域/` 划分 8 个领域，笔记内部用 `tags` 和 `aliases` 细分；`10_项目/项目名/` 一个项目一个文件夹
- **三层笔记类型**：永久笔记（`20_笔记/`，反复翻看延伸）、项目日记（`10_项目/`，记录过程，写完不改）、速查资源（`30_资源/`，纯命令/链接/模板）
- **两层任务管理**：`10_项目/任务总控.md` 看全局仪表盘，`项目名/任务清单.md` 看单项目待办
- **一次对话 = 一套笔记**：操作过程 → 日记，理解收获 → 永久笔记（一个概念一篇），互不混淆
- **大胆链出去**：每篇笔记至少一个 `[[wikilink]]`，空链接完全合法，先链后建

## 模板

**永久笔记**：

```markdown
---
aliases: []
created: YYYY-MM-DD
tags: []
---

# 主题（一句话概括）

（正文第一句直接说清这个概念。）

## 关键点
- 

## 怎么用
（贴代码，没有可省略）

## 关联
- [[]]
```

**项目日记**：

```markdown
---
tags: [diary]
created: YYYY-MM-DD
---

# YYYY-MM-DD 做了什么

## 操作步骤
## 踩坑记录
## 关联
```

**命名规范**：永久笔记 `主题.md`，日记 `YYYY-MM-DD_做了什么.md`，速查 `XX速查.md`。

**Properties 规范**：`tags` 用 YAML list 不加 `#`，用复数 `tags` 而非 `tag`（已废弃），`aliases` 放别名，属性名小写英文，标签不超过 3 个。

## 三技能联动工作流

三个 Claude 技能形成学习闭环：

```
开始会话 (/session)
    │  读取任务清单，展示进度，推荐下一个任务
    ▼
学习 / 工作中
    │  可穿插 /add-task（临时加任务）、/obsidian-note（过程记笔记）
    ▼
结束会话 (/session)
    │  分析成果 → 确认任务完成 → 更新清单 → 自动调用 /obsidian-note
    ▼
笔记产出（/obsidian-note）
    日记 + 永久笔记 + 技能清单维护
```

| 技能 | 触发 | 职责 |
|------|------|------|
| `session` | 开始/结束关键词 | 会话管理、进度追踪、成果确认 |
| `add-task` | 添加任务关键词 | 任务创建、归属判断、跨项目复制 |
| `obsidian-note` | 仅 `/obsidian-note` | 笔记整理、知识沉淀、清单维护 |

`30_资源/Claude技能清单.md` 存放完整技能目录。
