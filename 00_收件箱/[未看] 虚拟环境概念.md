# 虚拟环境与 Conda

虚拟环境让每个项目拥有独立的 Python 依赖，避免不同项目的包版本冲突。Conda 是 Python 生态中最常用的环境管理工具之一。

## 关键点

- **为什么需要虚拟环境**：不同项目可能依赖不同版本的 Python 或第三方库。共用一套环境容易出现版本冲突、升级一个项目影响另一个项目、难以复现运行环境
- **Conda 是什么**：包管理 + 环境管理工具，适合数据科学、机器学习和多版本环境管理。常见发行版有 Anaconda（重量级）、Miniconda（轻量）、Miniforge（社区）
- **一个项目一个环境**：每个项目对应独立环境，环境名有明确含义（如 `nlp_env`），项目完成后及时导出 `environment.yml`
- **Conda vs venv**：Conda 管理 Python 环境 + 非 Python 依赖，包来自 Conda 仓库；venv 只管 Python 虚拟环境，包来自 PyPI。科学计算用 Conda，轻量项目用 venv
- **优先 conda install**：Conda 装不到的再用 pip，尽量不要在同一个环境里频繁混用

## 怎么用

```bash
# 创建环境
conda create -n myenv python=3.11

# 激活 / 退出
conda activate myenv
conda deactivate

# 导出环境
conda env export > environment.yml

# 从文件恢复
conda env create -f environment.yml
```
