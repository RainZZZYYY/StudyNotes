# 阅读状态追踪

永久笔记的三态阅读追踪系统，用**文件名前缀 + 正文勾选框**双重标识笔记的阅读状态。

## 关键点

- **三态设计**：未看（完全没读）→ 待看（开头已确认）→ 已看（两头确认完毕，移入永久笔记区）
- **双重标识**：文件名前缀（`[未看]` / `[待看]` / 无前缀）和正文两个勾选框（开头 `[ ]` + 结尾 `[ ]`）互相印证，任一维度都可独立判断状态
- **位置联动**：未看和待看在 `00_收件箱/`，已看移入 `20_笔记/` 对应领域
- **每个状态跃迁需显式规则**：新建→未看、未看→待看、待看→已看，以及内容更新时的状态重置，四条规则各司其职
- **更新笔记时重置结尾**：追加内容后结尾勾选框重置为 `[ ]`，防止「内容变了但标记没变」的假确认
- **技能只检测不修改**：技能永远不写入 `[x]`，勾选动作仅由用户手动完成。更新笔记时如无勾选框则插入 `[ ]`，检测现有状态后仅执行重命名

## 状态判定表

| 文件名 | 开头 | 结尾 | 状态 | 位置 |
|--------|------|------|------|------|
| `[未看] 主题.md` | `[ ]` | `[ ]` | 未看 | `00_收件箱/` |
| `[待看] 主题.md` | `[x]` | `[ ]` | 待看 | `00_收件箱/` |
| `主题.md` | 无 | 无 | 已看 | `20_笔记/领域/` |

## 跃迁规则

- **新建 → 未看**：文件名加 `[未看]` 前缀，插入开头和结尾两个 `[ ]`，放入收件箱
- **未看 → 待看**：勾选开头为 `[x]`，重命名 `[未看]` → `[待看]`，留在收件箱
- **待看 → 已看**：去除文件名前缀、去除两个勾选框，移动到 `20_笔记/`
- **内容更新**：保留开头勾选框状态，重置结尾为 `[ ]`，根据开头状态重命名，移回收件箱
