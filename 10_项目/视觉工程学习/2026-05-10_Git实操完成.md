# 2026-05-10 Git 实操完成

## 操作步骤

- 在 WSL2 中配置 Git 用户名和邮箱
- `git init` 初始化 vision-engineering-learning 仓库，分支改名 `main`
- 创建 `.gitignore` 排除 `.venv/`
- 完成第一次 commit：hello_python.py + .gitignore
- 安装 GitHub CLI (`gh`)，遇到认证问题
- 排查 GitHub 连接超时：发现 Windows hosts 文件将 `github.com` 指向 `127.0.0.1`
- 修改 Windows hosts 后 HTTPS 仍间歇超时
- 生成 SSH 密钥（ed25519），配置到 GitHub
- `gh repo create` 创建远程仓库并推送
- 将 remote 从 HTTPS 切换为 SSH
- 创建 README.md
- 推送成功，GitHub 页面可见
- 额外：将 StudyNotes 笔记仓库独立建立并推送到 GitHub

## 踩坑记录

- **WSL2 连不上 GitHub**：Windows hosts 被某工具污染，所有 GitHub 域名指向 `127.0.0.1`，需要管理员权限编辑 `C:\Windows\System32\drivers\etc\hosts` 注释掉
- **HTTPS 推送超时**：WSL2 下 HTTPS 连接 GitHub 不稳定，改用 SSH 后稳定
- **gh auth 报 read:org 缺失**：生成 token 时需要同时勾选 `repo`、`read:org`、`workflow` 三项
- **第一次 push 报 upstream 缺失**：新分支需要 `git push --set-upstream origin main` 建立本地与远程的对应关系
