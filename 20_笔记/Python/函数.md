# 函数

函数是把一段可复用的代码封装起来、按需调用的方式。提高代码复用率，让程序结构更清晰，便于测试和维护。

## 关键点

**定义与调用**

```python
def greet(name):
    """文档字符串：描述用途、参数和返回值"""
    return f"你好，{name}！"

greet("Alice")
```

**参数类型**

| 类型 | 语法 | 说明 |
|------|------|------|
| 位置参数 | `def func(a, b):` | 按顺序传参 |
| 默认参数 | `def func(name, age=18):` | 必须放在位置参数后面 |
| 关键字参数 | `func(name="Tom", age=20)` | 按参数名传，顺序可变 |
| `*args` | `def func(*args):` | 接收多个位置参数，存为元组 |
| `**kwargs` | `def func(**kwargs):` | 接收多个关键字参数，存为字典 |
| 仅位置 | `def func(a, b, /):` | `/` 前只能按位置传 |
| 仅关键字 | `def func(*, name, age):` | `*` 后只能按关键字传 |

**返回值**：`return` 返回结果，没有 `return` 默认返回 `None`。返回多个值本质是返回一个元组。

**作用域**：遵循 LEGB 规则 —— Local → Enclosing → Global → Built-in。在函数内修改全局变量用 `global`，修改外层函数变量用 `nonlocal`。

**函数是一等对象**：可以赋值给变量、作为参数传递、作为返回值返回。

**闭包**：内部函数引用了外部函数变量，且外部函数返回了内部函数。

```python
def make_adder(x):
    def adder(y):
        return x + y
    return adder
```

**装饰器**：不修改原函数代码的前提下增加额外功能。`@decorator` 等价于 `func = decorator(func)`。

**lambda**：适合简短函数 `lambda x, y: x + y`，只能写单个表达式，复杂逻辑优先用 `def`。

## 常见易错点

- **不要用可变对象做默认参数**：`def func(lst=[])` 会导致多次调用共享同一个列表。用 `lst=None` + 内部判断。
- **位置参数必须在关键字参数前面**：`func(1, c=3)` 对，`func(a=1, 2)` 错。
- **不可变对象在函数内修改不影响外部**（int、str、tuple），可变对象原地修改会影响外部（list、dict、set）。
