# Claude 技能清单

三个自定义技能形成学习闭环。

## session

- **触发**："开始学习""继续学习""开始工作""今天做" → 开始会话；"收工""结束学习""今天做完了""先这样" → 结束会话
- **功能**：管理学习会话的始终。开始时指定项目并读取任务进度，推荐下一个任务；结束时扫描对话分析成果，逐任务确认完成状态，更新任务清单，自动调用 obsidian-note 整理笔记
- **流程**：开始 → 确定项目 → 展示进度 + 推荐任务 → 写会话标记 → 自由工作 → 结束 → 扫描对话 → 逐任务确认 → 更新清单 → 调用 obsidian-note
- **建议场景**：每次开始学习时说"开始学习"，结束时说"收工"，让技能自动管理进度和笔记

## add-task

- **触发**："添加任务""新建任务""加一个任务""创建任务""我要学""我要做"
- **功能**：根据一句话描述创建任务条目，自动判断归属（插入已有学习项目、新建截止日期项目、跨项目复制），由大模型决策最佳插入位置，自动维护序号。按项目类型选用不同模板：学习类有「学习内容·理论点·产出」，截止日期类有「任务描述·交付物·截止日期」
- **流程**：理解意图 → 判断归属（学习/截止日期/跨项目复制）→ 决定插入阶段和位置 → 选用模板生成条目 → 检测重复 → 更新任务清单和总控
- **建议场景**：突然想学新东西时说"我要学 Docker"，或接到一个截止日期项目时说"周五前完成缺陷检测 demo"

## obsidian-note

- **触发**：`/obsidian-note`（仅主动调用，不响应对话关键词）
- **功能**：将对话内容自动拆分为项目日记和永久笔记，先搜索已有笔记决定新建还是更新，避免重复建笔记；同时维护任务清单和技能清单
- **流程**：扫描对话 → 拆分操作过程和理解收获 → 搜已有笔记（新建/更新/跳过）→ 写笔记 → 双向链接 → 维护任务清单 → 维护技能清单 → 汇总确认 → 同步 README 进度 → Git 自动提交
- **建议场景**：完成一个学习任务后说 `/obsidian-note`、讨论出重要结论后沉淀知识、踩坑后保存经验、创建或修改技能后更新技能清单
