# Claude 技能清单

五个自定义技能形成学习闭环。

## session

- **触发**："开始学习""继续学习""开始工作""今天做" → 开始会话；"收工""结束学习""今天做完了""先这样" → 结束会话
- **功能**：管理学习会话的始终。开始时指定项目并读取任务进度，推荐下一个任务；结束时扫描对话分析成果，逐任务确认完成状态，更新任务清单，自动调用 obsidian-note 整理笔记
- **流程**：开始 → 确定项目 → 展示进度 + 推荐任务 → 写会话标记 → 自由工作 → 结束 → 扫描对话 → 逐任务确认 → 更新清单 → 调用 obsidian-note
- **建议场景**：每次开始学习时说"开始学习"，结束时说"收工"，让技能自动管理进度和笔记

## add-task

- **触发**："添加任务""新建任务""加一个任务""创建任务""我要学""我要做"
- **功能**：根据一句话描述创建任务条目，自动判断归属（插入已有学习项目、新建截止日期项目、跨项目复制），由大模型决策最佳插入位置，自动维护序号。按项目类型选用不同模板：学习类有「学习内容·理论点·产出」，截止日期类有「任务描述·交付物·截止日期」
- **流程**：理解意图 → 判断归属（学习/截止日期/跨项目复制）→ 决定插入阶段和位置 → 选用模板生成条目 → 检测重复 → 更新任务清单和总控
- **建议场景**：突然想学新东西时说"我要学 Docker"，或接到一个截止日期项目时说"周五前完成缺陷检测 demo"

## obsidian-note

- **触发**：`/obsidian-note`（仅主动调用，不响应对话关键词）
- **功能**：将对话内容自动拆分为项目日记和永久笔记，先搜索已有笔记决定新建还是更新；永久笔记三态追踪（未看→待看→已看），文件名前缀 + 正文勾选框双重标识；自动 push-skills
- **流程**：扫描对话 → 检查学习工作文档 → 搜已有笔记（新建/更新/跳过）→ 写笔记 → 应用阅读状态追踪 → 双向链接 → 维护任务清单 → 维护技能清单 → 汇总确认 → 同步 README → Git 提交 → 自动 push-skills
- **建议场景**：完成学习任务后整理产出、讨论出重要结论后沉淀知识、踩坑后保存经验

## learn

- **触发**：`/learn`（仅主动调用）
- **功能**：交互式学习助手。两种模式：`/learn 任务编号` 从任务清单读取知识点逐项推进，`/learn 自由内容` 快速了解任意知识。知识点标注 🔴🟡🟢 掌握等级（结合学习画像+联网搜索岗位相关度），按等级决定阶段深度；概念解释阶段联网搜索补充实践细节；生成学习工作文档（40_中间产物/当前学习.md）记录进度
- **流程**：读取任务+画像+断点检查 → 提取知识点+联网定级 → 确认 → 创建学习文档 → 逐知识点逐阶段推进（前置关联→概念解释含搜索→代码实例→动手测试）→ 穿插回顾 → 阶段间提示 → 完成后询问是否标记完成
- **建议场景**：按任务清单推进系统学习时说 `/learn 3｜Python｜基础语法复习`、临时想快速了解某个概念时说 `/learn 什么是装饰器`

## push-skills

- **触发**：`/push-skills`（仅主动调用）
- **功能**：检测 `~/.claude/skills/` 下的文件变更，自动检查并更新 README（新增/修改/删除技能时同步技能列表），生成提交信息，推送到 GitHub
- **流程**：检测变更 → 检查 README 是否需同步 → 展示变更摘要 → 用户确认 → 生成提交信息 → commit + push → 展示结果
- **建议场景**：创建或修改技能后说 `/push-skills`，一站式完成 README 同步和推送
